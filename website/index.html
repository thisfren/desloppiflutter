How it works

Desloppify combines mechanical detection (dead code, duplication, complexity) with subjective LLM review (naming, abstractions, module boundaries). It scans your codebase, builds a prioritized plan, then works through a fix loop — one issue at a time. State persists across sessions so it chips away over multiple runs, and the scoring is designed to resist gaming. A score above 98 should correlate with a codebase a seasoned engineer would call beautiful.

From vibe coding to vibe engineering

Vibe coding gets things built fast. But the codebases it produces rot in ways that are hard to see — abstractions drift, naming degrades, error handling diverges. It works, but working in it gets worse.

LLMs are good at spotting this now, if you ask the right questions. The bet: an agent with the right framework can hold a codebase to the standard that used to require a senior engineer paying close attention over months.
